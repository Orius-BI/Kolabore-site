Read each of the 8 section files and apply the following audit checks and fixes:

**Check 1: Motion import path**
For every file that imports motion:
- WRONG pattern: `import { motion } from 'motion/react'` or `import * as m from 'motion/react'`
- CORRECT pattern: `import * as m from 'motion/react-m'`
- If wrong: replace the import line with the correct one

**Check 2: HeroSection uses animate, not whileInView**
HeroSection is above the fold — the element is immediately visible on page load. Using whileInView here means the animation never fires until the user scrolls away and back. Correct pattern:
```tsx
<m.div
  initial={{ opacity: 0, y: 24 }}
  animate={{ opacity: 1, y: 0 }}      ← animate (not whileInView)
  transition={{ duration: 0.7, ease: 'easeOut' }}
  ...
>
```
If the current file uses whileInView on the hero content block, change it to animate.

**Check 3: All below-fold sections use whileInView with viewport once:true**
For every section other than HeroSection, verify all m.div elements that should animate have:
- `whileInView={{ opacity: 1, y: 0 }}`
- `viewport={{ once: true, amount: 0.1 }}` (or similar threshold)
- `initial={{ opacity: 0, y: 24 }}` (or matching variant)
If missing, add. The `once: true` is mandatory — prevents re-triggering when user scrolls back up.

**Check 4: ContactSection and TeamPreviewSection**
- ContactSection: does not need motion on the form itself (form submission is interactive, adding scroll animation can interfere). If motion was added to the form, remove it. The SectionHeading may have a simple fade if consistent with other sections, but the form fields should not individually animate.
- TeamPreviewSection: the empty-state guard (`return null`) means no motion is ever triggered. If the populated state card grid has motion, verify it uses the standard stagger pattern.

**Check 5: Confirm 'use client' on all files with m.* usage**
Every file that imports from 'motion/react-m' must have `'use client'` as its very first line. Verify and add if missing.

After all fixes, run the grep verification checks listed in acceptance_criteria.
