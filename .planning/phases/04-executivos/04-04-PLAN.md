Person schema shape (Google spec):
```json
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "Carlos Ferreira",
  "jobTitle": "Sócio",
  "description": "Bio text here.",
  "image": "https://kolabore.com.br/images/team/carlos-ferreira.webp",
  "worksFor": {
    "@type": "Organization",
    "name": "Kolabore"
  }
}
```

Next.js static export constraint:
- Cannot use next/head in App Router
- Use a `<script type="application/ld+json">` tag directly in the page JSX
- For multiple schemas (one per person), wrap them in a JSON-LD array OR inject one script tag per person
- metadataBase is configured — absolute image URLs must use process.env.NEXT_PUBLIC_SITE_URL or hardcode the production domain

  <action>
    Update src/app/executivos/page.tsx to inject JSON-LD Person schema for each executive.

    **Implementation approach:**
    Since this is an App Router page (Server Component), use `<script>` tags directly in the JSX return — this is the correct Next.js App Router pattern for JSON-LD:

    ```tsx
    // At the bottom of the page return, before closing </main>:
    {executives.map((exec) => (
      <script
        key={`jsonld-${exec.id}`}
        type="application/ld+json"
        dangerouslySetInnerHTML={{
          __html: JSON.stringify({
            "@context": "https://schema.org",
            "@type": "Person",
            "name": exec.name,
            "jobTitle": exec.title,
            "description": exec.bio,
            "image": `https://kolabore.com.br${exec.photo}`,
            "worksFor": {
              "@type": "Organization",
              "name": "Kolabore",
              "url": "https://kolabore.com.br"
            }
          })
        }}
      />
    ))}
    ```

    **Important notes:**
    - Use `dangerouslySetInnerHTML` — this is correct and safe for JSON-LD (data is not user input)
    - Image URL must be absolute: prepend `https://kolabore.com.br` to exec.photo
    - If executives array is empty, the map produces nothing — this is fine
    - Do NOT add 'use client' to page.tsx for this — script tags work in Server Components
    - The script tags must be placed inside the `<main>` element (not outside)

    After writing, run build:
    `npm run build`

    Then verify the schema is present in the built HTML:
    `grep -l "application/ld+json" out/executivos/index.html`
    `grep -c '"@type": *"Person"' out/executivos/index.html`
    (should return the count of executives)
