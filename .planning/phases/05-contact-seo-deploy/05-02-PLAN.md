```typescript
export const metadata: Metadata = {
  metadataBase: new URL("https://kolabore.com.br"),
  title: {
    default: "Kolabore | Consultoria Executiva",
    template: "%s | Kolabore",   // page title appends " | Kolabore"
  },
  description: "Consultoria executiva de alto valor...",
  openGraph: {
    type: "website",
    locale: "pt_BR",
    url: "https://kolabore.com.br",
    siteName: "Kolabore",
  },
}
```
IMPORTANT — shallow merge pitfall: If any page exports openGraph with even one key, it REPLACES the root openGraph entirely. The og:image from opengraph-image.jpg file convention is immune to this (file-based metadata has higher priority than config). Using the file convention means pages do NOT need to repeat the image in their openGraph config.
