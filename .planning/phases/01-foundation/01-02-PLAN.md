    Replace the entire contents of `src/styles/globals.css` with the following. Do NOT keep any `@tailwind base`, `@tailwind components`, or `@tailwind utilities` directives — those are Tailwind v3 syntax. Tailwind v4 uses a single `@import "tailwindcss"`.

    ```css
    /* src/styles/globals.css */
    /* Tailwind v4 — single import replaces @tailwind base/components/utilities */
    @import "tailwindcss";

    /* ─── Static design tokens ───────────────────────────────────────────────── */
    /* Tailwind emits these as CSS custom properties and generates utility classes */
    @theme {
      /* Brand colors */
      --color-ink:        #0d0d0d;    /* near-black — primary bg, body text */
      --color-carbon:     #1a1a1a;    /* card surfaces */
      --color-slate:      #2c2c2c;    /* borders, subtle dividers */
      --color-silver:     #8a8a8a;    /* muted text, eyebrow labels */
      --color-mist:       #f5f5f4;    /* warm light backgrounds */
      --color-white:      #ffffff;
      --color-gold:       #b8933f;    /* accent — CTAs, highlights — use sparingly */
      --color-gold-hover: #d4ad6b;    /* hover/focus state for gold */

      /* Type scale */
      --text-display: 3.5rem;         /* hero headline */
      --text-h1:      2.5rem;
      --text-h2:      1.875rem;
      --text-h3:      1.375rem;
      --text-body:    1rem;
      --text-small:   0.875rem;
      --text-label:   0.75rem;

      /* Spacing */
      --spacing-section:    5rem;     /* py-20 equivalent */
      --spacing-section-lg: 7rem;     /* py-28 — hero section */

      /* Radius — institutional, not rounded */
      --radius-card:   0.25rem;
      --radius-button: 0.125rem;
    }

    /* ─── Font tokens — reference next/font CSS variables ───────────────────── */
    /* MUST use @theme inline (not @theme) so Tailwind references the already-    */
    /* present CSS variables from next/font rather than emitting new ones.        */
    /* Variable names here MUST match exactly what is set in src/app/layout.tsx: */
    /*   Inter({ variable: "--font-inter" })                                      */
    /*   Cormorant_Garamond({ variable: "--font-cormorant" })                     */
    @theme inline {
      --font-sans:    var(--font-inter);
      --font-display: var(--font-cormorant);
    }

    /* ─── Base styles ────────────────────────────────────────────────────────── */
    body {
      background-color: var(--color-ink);
      color: var(--color-mist);
      font-family: var(--font-sans);
      -webkit-font-smoothing: antialiased;
      -moz-osx-font-smoothing: grayscale;
    }
    ```

    CRITICAL checks before saving:
    1. The file starts with `@import "tailwindcss"` — NOT `@tailwind base`
    2. Static color/type/spacing tokens are inside `@theme { }` (no `inline` keyword)
    3. Font tokens (`--font-sans`, `--font-display`) are inside `@theme inline { }` (WITH `inline` keyword)
    4. The variable names `var(--font-inter)` and `var(--font-cormorant)` must match exactly what Plan 1.3 will set via `next/font/google`. These names are the contract between this file and layout.tsx.

    After writing the file, run the build to verify no CSS errors:
